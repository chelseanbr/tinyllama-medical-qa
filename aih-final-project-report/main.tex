\begin{abstract}
Access to accurate, understandable medical information is essential, especially for laypersons without clinical expertise. This project explores how to improve layperson medical question answering (QA) by generating multiple-choice questions (MCQs) from the MedQuAD dataset and fine-tuning TinyLlama using a lightweight, parameter-efficient method (LoRA). Through rejection sampling and few-shot Chain-of-Thought prompting, a small MCQ dataset was curated. Fine-tuning led to some gains over baseline TinyLlama and MedAlpaca models, demonstrating the potential of focused domain adaptation even on smaller models.
\end{abstract}

\section{Introduction}
Medical QA remains a tough challenge for large language models (LLMs). Medical language is complex, and layperson users require responses that are both medically accurate and easy to understand. Many general-purpose LLMs are not reliably tuned for this domain, leaving significant gaps in accessibility to trusted medical information. To address this, I developed a pipeline to generate high-quality MCQs from the MedQuAD dataset and fine-tuned TinyLlama to better handle layperson medical QA. Improving accessibility to medical QA supports broader public health goals by making accurate, understandable answers more available to everyone.

\section{Related Work}
Ben Abacha and Demner-Fushman~\cite{BenAbacha-BMC-2019} developed an entailment-based framework for medical QA, focusing on ensuring that answers are both relevant and well-justified. Their work led to the creation of the MedQuAD dataset, which forms the foundation for the question-answer pairs used in this project. Yuan et al.~\cite{yuan2023scaling} proposed an offline method where a pre-trained LLM is guided through in-context learning to generate a dataset of high-quality question, reasoning, and answer triples. This idea of using model-generated reasoning traces to enhance training data quality influenced my approach, where I incorporated Chain-of-Thought prompting during MCQ generation to encourage more structured, explainable answers. Zhu et al.~\cite{zhu2024cura} demonstrated how synthetic QA data generation, when carefully controlled, can boost medical QA model performance. This project builds on these ideas by combining curated QA generation with efficient fine-tuning for a small, adaptable model.

I began by sampling 3,000 QA pairs from the MedQuAD dataset. A custom few-shot prompt (Figure~\ref{fig:mcqprompt}) with Chain-of-Thought (CoT) examples was designed to guide MCQ generation. Few-shot prompting helped the model produce well-structured questions with reasoning steps.

\subsection{Fine-Tuning TinyLlama with LoRA}
TinyLlama was fine-tuned using LoRA (Low-Rank Adaptation), which inserts small trainable matrices into attention layers instead of updating the full model. This enables efficient adaptation with limited resources.

Fine-tuning parameters:
\begin{itemize}
    \item LoRA Rank: 8
    \item LoRA Alpha: 32
    \item Optimizer: AdamW, learning rate 1e-3
\end{itemize}

The model was trained using supervised learning to predict the correct MCQ answer label.

\section{Conclusion}
This project developed a full pipeline for improving layperson medical QA by combining rejection-sampled MCQ generation with efficient LoRA fine-tuning on TinyLlama. Results showed some accuracy gains despite working with very limited generated data and small model size. 

In future work, I plan to scale fine-tuning to larger models like LLaMA-2 and Mistral, explore open-ended medical QA tasks, apply reinforcement learning (RLHF) for better question generation, and benchmark across more diverse medical QA datasets.
